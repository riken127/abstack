# abstack

`abstack` is a declarative DSL compiler that unifies image build definition and service instantiation.

One `.abs` source file compiles into:
1. `Dockerfile.<service>` per service
2. `docker-compose.generated.yml` for service orchestration

## Why

The v0.2 model treats compose as a projection of `service`, not as a separate authoring language.

- `template` defines reusable Docker build logic.
- `service` instantiates template(s) with runtime overlays.

## Build and Test

```bash
cmake --preset native
cmake --build --preset native
ctest --test-dir build/native --output-on-failure
```

## Run

```bash
./build/native/abstack samples/unified.abs --out-dir generated
```

Outputs:
- `generated/Dockerfile.api`
- `generated/Dockerfile.db`
- `generated/docker-compose.generated.yml`

## LSP

Use `clangd` with the compile database generated by the native preset (`build/native`).
