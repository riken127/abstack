# abstack v0.3.0 Release Notes

Release date: 2026-03-22

## Summary

v0.3.0 advances abstack from a single-template service model to a composable service model while improving Docker/Compose command fidelity and documentation maturity.

## Highlights

1. Services can now compose multiple template instantiations with repeated `use` statements.
2. `cmd` and `entrypoint` now support scalar and array forms.
3. Generated Dockerfile and compose output preserves command-array semantics.
4. Documentation expanded with a full language guide and examples catalog.
5. New sample files demonstrate quickstart, composition, and microservices usage.

## Compatibility Notes

1. Multi-use services are now valid and supported.
2. Runtime overlays are applied to the final lowered stage, so `use` order is meaningful.
3. `depends_on` now fails validation if a referenced service is not declared.

## Validation

Recommended pre-release checks:

```bash
cmake --preset native
cmake --build --preset native
ctest --test-dir build/native --output-on-failure
./build/native/abstack samples/microservices.abs --out-dir generated
```

## Suggested Next Targets

1. Optional explicit final-stage selection semantics for multi-use services.
2. Compose-level extras (`volumes`, `networks`, `healthcheck`) with strict declarative constraints.
3. Golden output snapshot tests for generated Dockerfile and compose artifacts.
