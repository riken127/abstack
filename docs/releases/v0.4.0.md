# abstack v0.4.0 Release Notes

Release date: 2026-03-22

## Summary

v0.4.0 focuses on CLI maturity: abstack now behaves like a utility suite rather than a single compile command.

## Highlights

1. New subcommand-oriented CLI:
   - `build`
   - `sync`
   - `fmt`
   - `docker` (minimal container ops helper)
   - `compose`
   - `tui` (optional)
2. Regex-driven sync workflows for dynamic compose generation from multiple `.abs` files.
   - Includes automatic per-file namespacing to prevent merge collisions.
3. Canonical formatter support with in-place update, stdout, and CI-style `--check` modes.
4. Thin docker compose integration via passthrough command execution.
5. Optional curses-backed TUI when available.

## Why This Scope

Direct full orchestration replacement is intentionally out of scope for abstack's compiler core.

This release ships a pragmatic compromise:

1. Keep compiler concerns first.
2. Offer convenient compose execution wrappers.
3. Preserve compatibility with native Docker Compose behavior.

## Validation

```bash
cmake --preset native
cmake --build --preset native
ctest --test-dir build/native --output-on-failure
./build/native/abstack build samples/unified.abs --out-dir generated
./build/native/abstack fmt samples --file-regex '.*\\.abs$' --check
./build/native/abstack sync --input-dir samples --file-regex '.*\\.abs$' --out-dir generated --clean
```

## TUI Build Notes

For vcpkg environments:

```bash
export VCPKG_ROOT=/path/to/vcpkg
export VCPKG_MANIFEST_FEATURES=tui
cmake --preset vcpkg
cmake --build --preset vcpkg
```

Then launch:

```bash
./build/vcpkg/abstack tui
```
